# Prompt Card: Генерация корпоративных политик

## 1. Общая информация
**Название:** Генерация корпоративных политик  
**Версия:** v1.0  
**Статус:** Active  
**Владелец:** HR / People Team  
**Дата создания:** 2026-04-08  
**Последнее обновление:** 2026-04-08  

---

## 2. Назначение
Промпт используется для подготовки черновиков корпоративных политик, регламентов и внутренних документов компании с учетом бизнес-контекста, понятности формулировок и базовой юридической аккуратности.

---

## 3. Когда использовать
- разработка новых корпоративных политик;
- обновление существующих регламентов;
- подготовка HR-документов перед юридической проверкой;
- унификация документов по группе компаний.

---

## 4. Целевые пользователи
- HR-специалисты;
- HRD;
- руководители функций;
- внутренние заказчики;
- юристы для финальной проверки.

---

## 5. Разбор по CLEAR

### Task
Подготовить текст корпоративной политики на основе заданных вводных.

### Context
Документ создается для компании, где важны:
- понятность для сотрудников;
- практическая применимость;
- снижение юридических рисков;
- баланс интересов бизнеса и работников.

### Input
Входные данные:
- тема политики;
- цель документа;
- тип бизнеса;
- размер компании;
- юрисдикция;
- особенности компании;
- ограничения;
- дополнительные вводные.

### Output
Результат:
- структурированный текст в Markdown;
- логичные разделы;
- нейтральный деловой стиль;
- текст, пригодный для доработки и согласования.

---

## 6. Входные параметры

| Параметр | Описание | Пример |
|---|---|---|
| `policy_topic` | Тема политики | Командировки |
| `policy_goal` | Цель документа | Унифицировать правила оформления |
| `business_type` | Тип бизнеса | B2B / производство |
| `company_size` | Размер компании | 300+ сотрудников |
| `jurisdiction` | Юрисдикция | РФ |
| `company_features` | Особенности компании | Группа компаний, филиалы |
| `constraints` | Ограничения | Без сложного юридического языка |
| `extra_input` | Дополнительные вводные | Исторически правила различались |
| `mode` | Режим работы | draft / production |

---

## 7. Шаблон промпта

```text
ROLE:
Ты — HR-директор и специалист по трудовым политикам компании с численностью 300+ сотрудников. Пишешь структурированно, понятно и по-деловому.

TASK:
Подготовь корпоративную политику на основе входных данных.

CONTEXT:
- Тип бизнеса: {{business_type}}
- Размер компании: {{company_size}}
- Юрисдикция: {{jurisdiction}}
- Особенности компании: {{company_features}}
- Ограничения: {{constraints}}

INPUT:
- Тема политики: {{policy_topic}}
- Цель документа: {{policy_goal}}
- Дополнительные вводные: {{extra_input}}

LOGIC:
1. Определи цель и область применения политики.
2. Предложи логичную структуру документа.
3. Опиши основные правила и порядок действий.
4. Отрази роли и ответственность участников процесса.
5. Добавь исключения, ограничения и спорные моменты, если они важны.
6. Проверь текст на ясность, логичность и отсутствие лишней воды.

OUTPUT FORMAT:
Верни результат в Markdown со структурой:

# {{policy_topic}}

## 1. Общие положения
## 2. Цель и задачи
## 3. Область применения
## 4. Основные правила
## 5. Ответственность сторон
## 6. Ограничения и особые условия
## 7. Заключительные положения

QUALITY CHECK:
- текст должен быть понятным;
- формулировки не должны быть избыточно сложными;
- стиль должен быть деловым, но человеческим;
- итог должен подходить как рабочий черновик для согласования.

MODE:
{{mode}}
